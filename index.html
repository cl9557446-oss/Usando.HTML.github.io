Módulo 2.2: Diseño de Software

El Diseño de Software es el proceso de transformar los requisitos del usuario en una forma que el programador pueda implementar. No se trata de escribir código inmediatamente, sino de crear el "plano" detallado que guiará todo el desarrollo.

1. Abstracción y Modularidad

Un buen diseño permite dividir un sistema complejo en piezas más pequeñas o módulos. Esto facilita que cada parte se diseñe de forma independiente, lo que reduce la carga cognitiva del equipo y permite que el sistema sea más fácil de entender.

2. Interfaz y Experiencia (UI/UX)

El diseño de software también contempla cómo interactuará el usuario con el sistema. Definir flujos de navegación claros y botones intuitivos es parte esencial del diseño técnico, asegurando que la herramienta sea útil y eficiente.

3. Documentación Técnica

Antes de programar, se utilizan diagramas (como UML) para representar la lógica. Esto asegura que todo el equipo esté en la misma sintonía en GitHub y que no haya confusiones sobre cómo deben conectarse las diferentes funciones del programa.

En conclusión, un diseño sólido ahorra tiempo y dinero, evitando errores costosos que suelen aparecer cuando se empieza a programar sin una planificación previa.
